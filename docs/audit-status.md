- 2026-07-22 — New (non-audit) finding, fix open for review (PR #113, `fix/ll2-crew-fetch-failure`): the live `/crew` endpoint is failing on every call in production (`ok:false`, empty roster, fresh `fetchedAt` each request) even though the same LL2 URLs return valid data from any non-Workers IP — the failure is specific to Cloudflare Workers egress. Leading suspect: LL2 throttles anonymous traffic at 15 requests/hour/IP (confirmed in The Space Devs' docs) and Workers egress IPs are shared across customers, with PR #96's don't-cache-failures behavior turning every visitor into another hit on the already-throttled shared IP. Cannot be reproduced or confirmed-fixed from a sandbox (the trigger is Workers egress itself) — after merge + auto-deploy, curl `/crew`: `sourceStatus` 429 → Ian creates a Space Devs API key and runs `npx wrangler secret put LL2_API_KEY` (persists across future deploys per Cloudflare docs); 403/blocked → fall back to a committed-JSON crew file refreshed by a scheduled workflow (same pattern as `/today`/`/capsules`, deliberately not built in this PR).
